# Системы обмена сообщениями

- __Сообщение (Message)__ — единица данных, передаваемая от отправителя к получателю. 
    Обычно содержит заголовки (метаданные) и тело (полезные данные).

- __Продюсер (Producer / Publisher)__ — компонент, который создаёт и отправляет сообщения.

- __Консьюмер (Consumer / Subscriber)__ — компонент, который получает и обрабатывает сообщения.

- __Брокер (Broker)__ — промежуточный сервис (например, RabbitMQ, NATS, Kafka), который принимает, 
    хранит, маршрутизирует и доставляет сообщения.

- __Очередь (Queue)__ — структура, хранящая сообщения в порядке поступления (FIFO). 
    Каждое сообщение обычно потребляется только одним консьюмером (если не настроено иное).

- __Топик (Topic)__ — именованный канал (или маршрут), через который продюсеры публикуют сообщения.
    В модели publish/subscribe сообщения из топика получают все подписчики.

## Цели:
- Асинхронность — продюсер не ждёт обработки сообщения.
- Слабосвязанность — компоненты общаются через брокера, не зная друг о друге.
- Масштабируемость — можно добавить больше консьюмеров в очередь.
- Устойчивость к пикам — очередь сглаживает неравномерную нагрузку.
- Надёжность — повторные попытки, сохранение на диск, DLQ (Dead Letter Queue).


## Модели обмена сообщениями
1) __Очередь (Point‑to‑point)__ 
    - Сообщение помещается в очередь.
    - Каждое сообщение забирается ровно одним консьюмером (если несколько слушают очередь, они делят нагрузку).
    - После успешной обработки сообщение удаляется из очереди.
2) __Обменник/Топик (Publish‑Subscribe)__
   - Сообщение публикуется в топик.
   - Все активные подписчики этого топика получают копию сообщения.
   - Брокер может сохранять сообщения для отключённых подписчиков (если включена устойчивость, durable subscription).
3) __Гибридная схема__

## Гарантии доставки
- __At-most-once__ - Сообщение может быть потеряно, но точно не доставлено дважды (без подтверждения)
- __At-least-once__ - Сообщение будет доставлено минимум один раз; возможны дубликаты (ожидание подтверждения)
- __Exactly-once__ - Сообщение доставляется строго один раз без потерь и дубликатов (механизмы идемпотентности, транзакции)

## Устойчивость к перезапуску
- __Persistent (durable)__ - хранение на постоянных запоинающих устройствах
- __Non‑persistent (transient)__ - хранение в оперативной памяти

## Приоритеты и задержки
- __Priority queue__ - очередь с приоритетами (вспомните бинарную кучу)
- __Delayed (scheduled) messages__ - задержка в очереди до определенного момента
- __TTL (Time‑To‑Live)__ - сообщения с ограниченным сроком пребывания в очереди (если не извлекли - удаляются)

## Пример обработки запросов
```python
# python_server.py
import asyncio
import nats
from nats.aio.msg import Msg

async def main():
    nc = await nats.connect("nats://localhost:4222")

    async def request_handler(msg: Msg):
        try:
            print(f"Получен запрос: {msg.data.decode()}")
            request_data = json.loads(msg.data.decode())
        
            response = {
                    "status": "success",
                    "received": request_data,
                    "timestamp": time.time()
            }
            response_json = json.dumps(response)
            
            await msg.respond(response_json.encode())
            print("Ответ отправлен")
        except json.JSONDecodeError as e:
            error_response = {"status": "error", "message": f"Invalid JSON: {str(e)}"}
            await msg.respond(json.dumps(error_response).encode())
            
        
    # Подписка на тему
    await nc.subscribe("request.json", cb=request_handler)
    print("Python‑сервер запущен, ожидает запросы на 'request.topic'")

    # Держим сервер активным
    try:
        await asyncio.Future()  # бесконечное ожидание
    except KeyboardInterrupt:
        await nc.close()

if __name__ == "__main__":
    asyncio.run(main())
```

### Клиент, работающий в режиме запрос-ответ

```java
import io.nats.client.*;
import io.nats.client.api.PublishAck;
import com.fasterxml.jackson.databind.ObjectMapper;
import com.fasterxml.jackson.databind.JsonNode;

import java.nio.charset.StandardCharsets;
import java.time.Duration;
import java.util.Base64;
import java.util.HashMap;
import java.util.Map;

public class JavaJsonClient {
    public static void main(String[] args) {
        String natsUrl = "nats://localhost:4222";
        String subject = "request.json";

        byte[] img = null;
        
        Map<String, Object> requestMap = new HashMap<>();
        requestMap.put("service", "resize");
        requestMap.put("image", Base64.getEncoder().encodeToString(img));

        try (Connection nc = Nats.connect(natsUrl)) {
            System.out.println("connected");

            ObjectMapper mapper = new ObjectMapper();
            String jsonRequest = mapper.writeValueAsString(requestMap);
            System.out.println("send JSON: " + jsonRequest.substring(0,30));

            Message reply = nc.request(subject, jsonRequest.getBytes(StandardCharsets.UTF_8),
                    Duration.ofSeconds(10));

            // Парсим JSON-ответ
            String jsonResponse = new String(reply.getData(), StandardCharsets.UTF_8);
            System.out.println("response " + jsonResponse);
        } catch (InterruptedException e) {
            Thread.currentThread().interrupt();
            System.err.println("Запрос прерван");
        } catch (Exception e) {
            e.printStackTrace();
        }
    }
}
```
